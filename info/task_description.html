You are standing next to a machine with two numeric inputs x and y, and a single numeric output. It is known that the machine evaluates an expression using x, y, the operators + â€“ * / and brackets. You should recover the expression by using the machine with inputs of your choice in several turns. x, y and the result are integers. The machine uses real division(/) in the expression (For example: 5/2=2). When dividing by zero, the result will evaluate to "ZeroDivisionError".

Equivalent expressions will be accepted by the grader. For this the machine evaluates your expression and the hidden expression several times, with random values.

In each step, your function gets a list with data from the past steps. Each element is a list of three numbers -- x(int), y(int) and output(float or "ZeroDivisionError").

The function should return a list with three elements -- your guess as an expression string, x and y.

Input: A list with data from the past steps.

Output: A list with three elements -- your guess as an expression string, x and y.

Example: 

checkio([])  # hidden expression "x+y"
checkio([[2, 2, 4]]) # ["x*y", 3, 3]
checkio([[2, 2, 4], [3, 3, 6]])  # ["x+y", 5, 5]
